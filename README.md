# 就业数据自动化处理工具

自动完成Excel数据表格的清洗、验证、合并与标注。

## 功能概述

本脚本自动执行以下处理流程：

| 阶段 | 功能 | 说明 |
|------|------|------|
| 一 | 企业信息匹配与修正 | 读取企业信息TXT，匹配单位名称，填充/修正信用代码和注册资本 |
| 二 | 业务规则审核 | 人数配额检查（每单位≤3人）、注册资本审核、个体工商户识别 |
| 三 | 数据合并 | 将审核通过的行合并到全校总表（带人数上限控制） |
| 四 | 结果输出 | 输出处理后书院表、更新后总表、出错学生信息表 |

## 环境要求

- Python 3.8+
- 依赖库：`pandas`、`openpyxl`

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备输入文件

在脚本所在目录创建 `input/` 文件夹，并将以下三个文件放入：

- **`input/书院待筛选表格.xlsx`** — 每天收到的新数据表格
- **`input/全校数据总表格.xlsx`** — 累积的汇总表
- **`input/所有企业信息.txt`** — 企业信息文本文件（**格式不限，随意放**，脚本会自动识别）

### 3. 运行脚本

```bash
# 使用默认路径（文件在 input/ 目录下）
python data_processor.py

# 指定自定义文件路径
python data_processor.py --new "input/书院待筛选表格.xlsx" \
                         --total "input/全校数据总表格.xlsx" \
                         --enterprise "input/所有企业信息.txt"
```

### 4. 查看输出

处理完成后，会在 `output/` 目录生成三个文件：

| 文件 | 说明 |
|------|------|
| `output/书院待筛选表格_已处理_YYYYMMDD.xlsx` | 处理后的书院表格（含红/黄颜色标注和原因列） |
| `output/全校数据总表格_更新后_YYYYMMDD.xlsx` | 合并后的最新总表（**不会覆盖原始总表**） |
| `output/出错学生信息_YYYYMMDD.xlsx` | 标黄行的学生信息汇总（仅当存在学生姓名列时生成） |

## 企业信息TXT格式说明

### 无需固定格式，随意放

脚本采用**智能解析引擎**，不需要你按特定格式整理企业信息。以下格式都能自动识别：

**格式1：标签多行式**
```
企业名称：北京星辰科技有限公司
统一社会信用代码：91110108MA01ABCD2X
注册资本：5000万元
经营状态：存续
```

**格式2：一行一个企业**
```
北京星辰科技有限公司 91110108MA01ABCD2X 5000 存续
上海海纳贸易有限公司 91310000MA07EFGH5Y 1000 存续
```

**格式3：混合分隔符**
```
广州致远咨询中心，信用代码：91440101MA09IJKL8Z，注册资本：30万元，状态：存续
```

**格式4：自由段落**
```
第一家公司是北京星辰科技有限公司，信用代码91110108MA01ABCD2X，注册资本5000万元，存续
第二家是上海海纳贸易有限公司，信用代码91310000MA07EFGH5Y，注册资本1000万，存续
```

> **原则**：只要文本中包含企业名称、信用代码、注册资本等信息就行，不需要关注排版格式。

### 脚本会自动提取哪些信息

| 字段 | 识别方式 |
|------|---------|
| 单位名称 | 自动识别"企业名称：xxx"标签，或包含"有限公司"/"分公司"等后缀的名称 |
| 社会信用代码 | 自动识别18位或15位信用代码，支持"信用代码：xxx"标签或裸码 |
| 注册资本 | 自动识别"注册资本：xxx万"标签或"xxx万元"格式 |
| 状态 | 自动识别"存续/注销/在营"等状态关键词 |

## Excel表格格式要求

### 书院待筛选表格（输入）
- 必须包含列：**单位名称**
- 建议包含列：**学生姓名**（用于生成出错学生信息表）
- 可选包含列：社会信用代码、注册资本（会被修正覆盖）

### 全校数据总表格（输入）
- 必须包含列：**单位名称**
- 建议包含列：结构尽量与书院表格一致（按列名匹配）

> **重要**：列名匹配严格基于表头文字。例如，如果总表中单位列名为"单位"而非"单位名称"，则需要修改保持一致。

## 审核规则一览

| 规则 | 触发条件 | 处理方式 |
|------|---------|---------|
| 企业未找到 | 单位名称不在企业信息字典中 | 行标红 |
| 信用代码错误 | 表格中信用代码与企业字典不一致 | 静默修正，行标黄，写明原因 |
| 人数超额 | 总表人数+本次上报 > 3 | 行标黄，写明当前人数和可增人数 |
| 注册资本不足 | 20万≤资本<50万，且上报>2人 | 行标黄，最多报2人 |
| 注册资本过少 | 0<资本<20万 | 行标黄，需老师确认 |
| 个体工商户 | 信用代码以"92"开头 | 行标黄，不能上报 |
| 合并名额满 | 合并时发现总表该单位已达上限 | 未合并行标黄，写明原因 |

## 常见问题

### Q: 脚本报"文件不存在"错误？
确认三个输入文件已在 `input/` 目录下。也可使用绝对路径指定文件位置。

### Q: 输出文件中的颜色在WPS中看不到？
本脚本使用 openpyxl 标准颜色格式。如使用WPS，建议用Microsoft Excel打开，或升级WPS至最新版本。

### Q: 如何处理分公司（注册资本为0）？
注册资本为0或空的企业，系统会自动跳过注册资本审核规则，仅进行人数配额检查和个体工商户检查。

### Q: 企业信息TXT解析不准怎么办？
智能解析不是100%完美。如果某个企业信息提取错误，可以检查TXT文件内容是否完整，或适当添加标签（如"企业名称：xxx"、"信用代码：xxx"）帮助识别。
